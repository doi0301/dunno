# DUNNO 개발 태스크 목록 (해커톤 4시간 기준)

## 사용 방법
- 우선순위는 P0 > P1 > P2 순서
- 기본 원칙: P0(MVP) 완료 전에는 P1/P2 진행 금지
- 각 태스크의 완료 기준(DoD)을 만족하면 체크

---

## P0. MVP 필수 태스크 (반드시 완성)

### 0-1. 프로젝트 초기 세팅
- [ ] 단일 정적 웹앱 구조 생성 (`index.html`, `style.css`, `app.js`)
- [ ] 모바일 뷰포트 메타 태그 및 기본 레이아웃 골격 구성
- [ ] 카테고리 상수(이모지/색상 맵) 정의

완료 기준 (DoD)
- [ ] 모바일 세로 화면에서 기본 레이아웃이 깨지지 않는다
- [ ] 브라우저에서 빌드 없이 바로 실행된다

### 0-2. 데이터 모델 및 저장소(localStorage)
- [ ] Item 스키마 구현: `id`, `name`, `location`, `category`, `quantity`, `memo`, `imageUrl`, `createdAt`
- [ ] localStorage 읽기/쓰기 유틸 구현
- [ ] 앱 시작 시 저장 데이터 로드 및 메모리 상태 초기화

완료 기준 (DoD)
- [ ] 새로고침 후에도 데이터가 유지된다
- [ ] 잘못된 데이터가 있어도 앱이 죽지 않고 기본값으로 복구된다

### 0-3. 물건 추가 UI/로직
- [ ] 하단 플로팅 + 버튼 구현
- [ ] 추가 폼(이름/위치/카테고리/수량/메모) 구현
- [ ] 입력 유효성 검사(이름, 위치, 카테고리 필수)
- [ ] 저장 시 목록 즉시 반영

완료 기준 (DoD)
- [ ] 유효한 입력으로 저장 시 카드가 즉시 생성된다
- [ ] 필수값 누락 시 사용자에게 명확한 안내가 보인다

### 0-4. 목록/그룹 렌더링
- [ ] 카드 리스트 렌더링 구현
- [ ] 카테고리별 그룹 헤더 렌더링 구현
- [ ] 카드에 이름/위치/수량/메모 표시

완료 기준 (DoD)
- [ ] 모든 아이템이 카테고리 그룹 안에 올바르게 표시된다
- [ ] 데이터가 없어도 빈 상태 UI가 자연스럽게 보인다

### 0-5. 이미지 썸네일 + 이모지 폴백
- [ ] 아이템 이름 기반 이미지 URL 생성 함수 구현(키 없는 무료 소스 우선)
- [ ] 카드 썸네일 `<img>` 렌더링
- [ ] 이미지 로드 실패 시 `onerror`로 카테고리 이모지 폴백

완료 기준 (DoD)
- [ ] 이미지가 로드되면 썸네일이 보인다
- [ ] 로드 실패 시 앱 기능 저하 없이 이모지로 대체된다

### 0-6. 검색/필터
- [ ] 상단 검색바 구현(이름/위치 동시 검색)
- [ ] 카테고리 칩 필터 구현(전체 + 개별 카테고리)
- [ ] 검색 + 카테고리 필터 동시 적용

완료 기준 (DoD)
- [ ] 입력 즉시 실시간 필터링된다
- [ ] 필터 조합 시 예상 결과와 일치한다

### 0-7. 수정/삭제
- [ ] 카드에서 수정 진입(모달/인라인 중 택1)
- [ ] 수정 저장 시 기존 아이템 업데이트
- [ ] 삭제 액션 구현(버튼 또는 스와이프)

완료 기준 (DoD)
- [ ] 수정 후 목록/저장소가 일관되게 갱신된다
- [ ] 삭제 후 카드가 즉시 사라지고 저장소에서도 제거된다

### 0-8. UI 완성도(모바일 + 깜찍 스타일 + 인터랙션)
- [ ] 둥근 카드, 파스텔 톤, 터치 친화 spacing 적용
- [ ] 아이템 추가 시 bounce 애니메이션
- [ ] 아이템 삭제 시 fade-out 애니메이션
- [ ] 빈 상태 일러스트/문구 구성

완료 기준 (DoD)
- [ ] 모바일에서 터치 영역이 충분하다
- [ ] 추가/삭제 애니메이션이 끊김 없이 동작한다

### 0-9. 수동 테스트 체크리스트
- [ ] 추가 → 새로고침 → 유지 확인
- [ ] 검색(이름/위치) 확인
- [ ] 카테고리 필터 확인
- [ ] 수정/삭제 확인
- [ ] 이미지 실패 상황(네트워크 차단 등)에서 이모지 폴백 확인

완료 기준 (DoD)
- [ ] PRD의 MVP Acceptance Criteria를 모두 만족한다

## P1. 시간 남으면 (SHOULD)

### 1-1. 그룹 기준 토글
- [x] 카테고리별 보기 ↔ 위치별 보기 토글

### 1-2. 온보딩/샘플 데이터
- [x] 첫 진입 시 안내 문구 또는 샘플 데이터 제공

### 1-3. 수량 빠른 조절
- [x] 카드 내 +/- 버튼으로 수량 증감

---

## P2. 선택 확장 (COULD, AI 1개만)

### 2-1. 자동 카테고리 추천
- [ ] 이름 입력 시 카테고리 제안

### 2-2. 자연어 검색
- [ ] "충전 관련 물건" 같은 질의 해석

### 2-3. 자연어 추가
- [ ] 문장 입력으로 필드 자동 파싱

주의
- [ ] P2는 MVP 완료 후 1개만 선택

---

## 배포 태스크 (Azure)

### 3-1. 배포 준비
- [ ] 배포 대상 선택(Azure Static Web Apps 권장)
- [ ] `azd init` 및 환경 설정

### 3-2. 배포 실행
- [ ] `azd up` 실행
- [ ] 공개 URL 접속 확인(모바일 포함)

### 3-3. 데모 리허설
- [ ] 데모 시나리오(빈 상태 → 3~4개 추가 → 검색/필터 → 수정/삭제) 점검
- [ ] 이미지 실패 폴백 시나리오까지 점검

완료 기준 (DoD)
- [ ] 외부 네트워크에서 URL 접속 및 핵심 기능 시연 가능

## 권장 작업 순서 (실행 플랜)
1. 0-1 ~ 0-4 (기본 뼈대)
2. 0-2 + 0-3 (저장/추가)
3. 0-5 (이미지 + 폴백)
4. 0-6 + 0-7 (검색/수정/삭제)
5. 0-8 (UI polish)
6. 0-9 (최종 점검)
7. 3-1 ~ 3-3 (배포/리허설)

- 00:30~01:30: 추가/목록/저장
- 01:30~02:10: 검색/필터/수정/삭제
- 02:10~02:40: 이미지 + 폴백
- 02:40~03:20: UI/애니메이션/빈 상태
- 03:20~03:40: 테스트/버그 수정
